# NQRustBackup Installer

A Rust TUI installer for the NQRustBackup server stack, modelled on the
[NQRust-MicroVM](https://github.com/NexusQuantum/NQRust-MicroVM) installer.
Same look and feel (ratatui + crossterm), same modular shape
(`installer/` logic + `ui/screens` + `ui/widgets`), tuned for the
Director + Storage + FileDaemon + Catalog + WebUI stack.

## Quick install (curl)

```bash
# Default: TUI if a TTY is attached, otherwise headless install with defaults.
curl -fsSL https://raw.githubusercontent.com/NexusQuantum/CDN-Backup/master/scripts/install.sh | sudo bash

# With flags (forwarded to `nqrustbackup-installer install`):
curl -fsSL https://raw.githubusercontent.com/NexusQuantum/NQRust-Backup/master/scripts/install.sh | \
    sudo bash -s -- --profile server-only --webui-port 9100 --dry-run
```

Knobs:
- `NQRB_REPO=<owner>/<repo>` — override the source repo (default `NexusQuantum/NQRust-Backup`).
- `NQRB_VERSION=v0.1.0` — pin a specific release tag (default `latest`).
- `NQRB_NO_RUN=1` — only download the binary to `/usr/local/bin/nqrustbackup-installer`; don't run it.

## Build from source

```bash
git clone https://github.com/NexusQuantum/NQRust-Backup
cd NQRust-Backup/installer
cargo build --release
# → target/release/nqrustbackup-installer  (~2.2 MB)
```

## Usage

### Interactive TUI (default)

```bash
sudo ./target/release/nqrustbackup-installer
# or explicitly
sudo ./target/release/nqrustbackup-installer tui
```

### Non-interactive (CI / scripted / over ssh)

```bash
sudo nqrustbackup-installer install \
    --source upstream-compat \
    --profile all-in-one \
    --webui-port 9100

# Other sources:   --source build-from-source | configure-only
# Other profiles:  --profile server-only | client-only
# Dry-run (prints commands without executing):  --dry-run
```

### Preview the plan without running

```bash
nqrustbackup-installer plan --source upstream-compat --profile all-in-one
```

## What it installs

| Phase | Action |
|-------|--------|
| 1 | `apt-get install` + NQRustBackup apt repo setup + package install |
| 2 | PostgreSQL catalog bootstrap (create\_bareos\_database / make\_bareos\_tables / grant\_bareos\_privileges) |
| 3 | Render NQRustBackup config fragments (FullRoot FileSet, EvalPool) into `/etc/bareos/bareos-dir.d/` |
| 4 | `systemctl enable --now` the director, storage, and filedaemon |
| 5 | Deploy WebUI (apache2 + bareos-webui conf + admin console profile) |
| 6 | `bconsole reload` to pick up fragments |
| 7 | Verify: systemd units, bconsole → DIR, catalog reachable, WebUI HTTP 200 |

## Install sources

- **`upstream-compat`** (default) — installs from `download.bareos.org`.
  NQRustBackup is config-compatible with these packages; config grammar
  and file layout match. **Verified end-to-end on Ubuntu 24.04 by the
  installer test run** (2026-04-25).
- **`build-from-source`** — runs `apt build-dep . && dpkg-buildpackage`
  in the repo root this installer lives in. Produces ~30 `.deb` files.
- **`configure-only`** — skip package install; just render configs and
  start services. For hosts with binaries already in place.

## Install profiles

- **`all-in-one`** — DIR + SD + FD + Catalog + WebUI on one host. For
  testing and small deployments.
- **`server-only`** — DIR + SD + Catalog, no FD, no WebUI.
- **`client-only`** — FileDaemon only.

## Pre-flight checks

Before running, the installer verifies:

- OS is Debian/Ubuntu (warn otherwise, block only on `--source=upstream-compat`)
- Architecture is x86\_64
- Running as root OR sudo is available
- Install dir is writeable (~2 GiB free)
- Storage dir is writeable (~20 GiB free)
- systemd is present
- `download.bareos.org` resolves

Each check is either **ok**, **warn**, or **fail**. Any `fail` blocks proceeding.

## WebUI

After install, the WebUI is reachable at
`http://<host>:<webui-port>/bareos-webui/` — default port **9100**. Log in
with the `admin` console profile; the password is in
`/etc/bareos/bareos-dir.d/console/admin.conf` (autogenerated from
`admin.conf.example` on first install).

## Verified end-to-end

Installer test run 2026-04-25 against a fresh Ubuntu 24.04 VM on KubeVirt:

- ✓ 5/5 systemd units active (bareos-director, bareos-storage,
  bareos-filedaemon, postgresql, apache2)
- ✓ All four Bareos ports listening (9100, 9101, 9102, 9103)
- ✓ `bconsole → status director` returns `1000 OK: bareos-dir Version: 25.0.4`
- ✓ WebUI HTTP 200
- ✓ Catalog query `select count(*) from version` returns 1

## When NQRustBackup publishes its own apt repo

Swap two URLs in `src/installer/deps.rs::configure_upstream_repo` and
re-build. The rest of the installer is unchanged: package names
(`bareos-director`, etc.) match, config grammar matches, systemd unit
names match.

## Project layout (mirrors NQRust-MicroVM)

```
src/
├── main.rs                 entry + clap CLI
├── app.rs                  state machine
├── theme.rs                colors / symbols
├── installer/              side-effect modules
│   ├── preflight.rs        OS/disk/net checks
│   ├── deps.rs             apt + repo setup
│   ├── source.rs           dispatch by source
│   ├── database.rs         PG catalog bootstrap
│   ├── config.rs           render /etc/bareos fragments
│   ├── services.rs         systemd enable/start/reload
│   ├── webui.rs            apache2 + webui wiring
│   ├── verify.rs           post-install checks
│   └── executor.rs         phase orchestration + sudo_run
└── ui/                     TUI
    ├── mod.rs
    ├── screens/            one file per screen
    └── widgets/            checklist, log_viewer, phase_progress, status_bar
```
